# 项目管理方案（总控）

## 1. 流程

看板流转：`Backlog -> Ready -> In Progress -> Review -> QA -> Done`

分支规范：
- `feat/*`
- `fix/*`
- `chore/*`

PR 规范：
- 必须关联任务卡
- 必须包含测试/验证结果
- 必须有变更摘要与风险说明

## 2. 节奏

- 每日两次汇报：中午进度 / 晚间验收
- 每两天一个里程碑验收
- 关键变更打 tag，支持快速回滚

## 3. 质量闸门（DoD）

1. Lint 通过
2. 单测通过
3. 状态机与结算幂等测试通过
4. Dashboard 功能验证通过（桌面 + 移动）
5. 文档同步更新

## 4. 风险控制

- 模块单 owner，避免 Claude/Codex 同文件冲突
- 关键模块（engine/db）先测后合并
- 规则变更必须同步 `rules.json` 与变更记录
