## 📊 进度控制方法

### 1. 看板管理（GitHub Projects）
- **列**：Backlog → Ready → In Progress → Review → QA → Done
- **卡片**：每个任务一张，包含：描述、负责人、验收标准、截止时间
- **更新频率**：每完成一个子任务立即移动卡片

### 2. 汇报机制
- **每日两次固定汇报**：中午 12:00、晚间 22:00
- **紧急阻塞**：立即通报（15分钟内）
- **汇报格式**：
  ```
  ## [日期] [时间] 汇报
  ✅ 已完成：...
  🚧 进行中：...（进度%）
  ⚠️  阻塞点：...（需要什么帮助）
  📅 下一步：...（预计完成时间）
  ```

### 3. 风险预警
- **红/黄/绿**状态灯：
  - 🔴 高风险：可能影响里程碑
  - 🟡 中风险：可能影响进度但可控
  - 🟢 低风险：正常推进
- **风险登记册**：docs/RISK_REGISTER.md 实时更新

### 4. 变更管理
- 任何范围变更必须更新 SOW 并获批准
- 技术决策变更必须记录 ADR
- 分支策略：feat/*、fix/*、chore/*，PR 必须关联任务卡
