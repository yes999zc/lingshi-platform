# Lingshi 项目最佳实践总结（项目收官）

## 一、项目治理最佳实践

1. **执行与管理分离（RACI）**
   - Codex / Claude：负责实现与修复
   - PM（Mac）：负责30分钟监控、调度、风险汇报
   - 效果：减少“没人盯盘”与任务漂移

2. **固定节奏监控 + 固定汇报模板**
   - 监控频率：每30分钟
   - 汇报模板：`已完成 / 进行中（负责人+ETA） / 阻塞 / 下一步`
   - 效果：决策信息稳定、可比较

3. **以“证据”为主的进展判定**
   - 必须附测试结果、构建结果、健康检查，不接受口头“应该好了”

---

## 二、工程质量最佳实践

1. **先稳测试链路，再做功能迭代**
   - 将测试脚本切到稳定链路（`build + node --test`）规避环境不稳定因素

2. **关键问题优先打穿（P0/P1优先）**
   - 先清 release blocker，再做体验优化
   - 用 gate 脚本做统一验收

3. **发布前最小必过清单**
   - `npm test` 全绿
   - `integration-gate` 全通过
   - `/health` 返回 `ok`
   - Dashboard 可构建并可访问

---

## 三、产品与文档最佳实践

1. **中英文双文档分离（不是混排）**
   - `README.md`（English）
   - `README.zh-CN.md`（中文）
   - 相互跳转，兼顾海外与国内读者

2. **场景文档单独沉淀**
   - 将应用场景、MVP、KPI、商业化写成独立文档，便于商务与实施复用

3. **宣传资料产品化**
   - 首发文案、7天传播节奏、渠道策略放入 `PROMOTION_KIT` / 执行清单

---

## 四、发布与运营最佳实践

1. **先完善仓库面，再公开**
   - README、LICENSE、Release Notes、Topics、Description 先齐备

2. **一次完成标准发布动作**
   - commit → push → tag → release → repo metadata

3. **公开后持续运营**
   - 每周追踪：Stars、Issues、PR、Demo UV、潜在线索

---
